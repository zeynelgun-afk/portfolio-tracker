## 4. Yapılan Değişiklikler

### 4.1 `docs/FMP_SKILL.md` (+209 / -38 satır)

#### Yeni `fmp_get` ve `fmp_get_retry` örneği (Standard Python Pattern bölümü)

İki ayrı exception sınıfı tanımlandı:

```python
class FmpRateLimit(Exception):
    """429 status veya body'de 'Limit Reach' — bekle ve tekrar dene."""

class FmpEndpointError(Exception):
    """404/400/Invalid API KEY/JSON parse hatası — retry anlamsız."""
```

`fmp_get` artık şu sıralamayla işliyor:

1. Network exception (`Timeout`, `ConnectionError`) → `FmpRateLimit` (geçici)
2. HTTP 429 / 500 / 502 / 503 / 504 → `FmpRateLimit`
3. HTTP 4xx (diğer) → `FmpEndpointError` (kalıcı)
4. JSON parse hatası (`ValueError` veya `JSONDecodeError`) → `FmpEndpointError`
5. Body'de `"Error Message"` → "Limit Reach" varsa `FmpRateLimit`, yoksa `FmpEndpointError`
6. Boş `[]` artık geçerli yanıt sayılıyor, retry yapılmıyor

`fmp_get_retry` ise sadece `FmpRateLimit` exception'ında retry yapıyor; `FmpEndpointError`'da hemen `None` dönüyor. Rate limit beklemesi 60 saniyeden başlıyor, sonraki denemeler 1.5x büyüyor (60s, 90s, 135s).

|---------------|---------------|

## 6. Etki Tahmini

### 6.1 Rate limit tasarrufu

Sabah portföy taraması (`scripts/morning_scan.py`) ve bilanço sonrası tarama (`skills/bilanco-sonrasi-us`) toplam yaklaşık 1.300 hisse için profil çağırıyor, bunların yaklaşık %5-8'i (yaklaşık 65-100 hisse) `news/stock` veya `earnings-calendar` boş dönüyor (haber yok / earnings yok). Eski retry mantığı bu 65-100 boş yanıt için ek 2 retry × 2s = en az 130-200 saniye boş bekleme yapıyordu, ek olarak 130-200 boşa rate-limit call'u harcıyordu.

Yeni mantık ile bu boşa harcama sıfıra indi. Pipeline çalışma süresinde tahminen %3-5 hızlanma, rate limit sayacında daha geniş güvenlik marjı bekleniyor.

### 6.2 Hata gözlemlenebilirliği

`agent/fmp_client.py` zaten `observability.log_fmp_call()` üzerinden `logs/events.jsonl`'e log basıyordu, ama "Limit Reach" body mesajı gizli kayboluyordu (silent failure / sessiz hata). Artık log'a düşüyor, haftalık raporda görünür hale geliyor.

### 6.3 Geliştirici deneyimi

Skill artık 13 alan tuzağı için tek başvuru kaynağı (canonical reference / kanonik referans). Yeni script yazarken `quote LIST döner mi`, `treasury-rates yüzde mi ondalık mı`, `historical newest-first mi` gibi sorulara doğrudan cevap var. Önceden bu bilgiler memory'de dağınıktı veya hiç yoktu.
